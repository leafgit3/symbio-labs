Implementation plan (next iteration)

Track A: Decision-slot enforcement
1) Extend agent turn contract with stance output:
   - stance: escalate | contain | monitor
   - optional confidence: 0-1
2) Require cycle runner to satisfy slot minimums:
   - at least one escalate candidate
   - at least one contain candidate
   - at least one monitor candidate
3) If slots are missing, use deterministic tie-break fill rules by role class.
4) Add stance provenance diagnostics so synthetic coverage is visible:
   - stance_source: organic | orchestrator_fallback
   - fallback_reason (when used)
   - forced_slots_count per cycle

Track B: Event promotion outside prose
1) Add an orchestration-level promotion function.
2) Promote active events by thresholds, not wording style:
   - stance imbalance
   - contradiction density
   - noise/trust thresholds
   - repeated unresolved signals over N cycles
3) Persist promoted event descriptors to world_state.active_events.

Track C: Salience stratification
1) Replace fixed 0.65 with computed salience score.
2) Inputs for salience score:
   - contradiction against recent feed
   - novelty vs recent memory
   - threshold relevance (event-linked cues)
   - public/private mismatch
3) Clamp to bounded range (for example 0.2-0.95) and store in agent_memories.salience.

Track D: Anti-convergence pressure and observability
1) Add anti-convergence pressure in synthesis when all stances collapse too early.
2) Add cycle-level diagnostics to summaries/history:
   - stance distribution
   - stance provenance distribution (organic vs fallback)
   - promoted_events_count
   - contradiction_score
3) Surface diagnostics in History details and Dashboard latest summary card.

Suggested PR slices

- History shows stance/event diagnostics for at least the latest 10 cycles.
