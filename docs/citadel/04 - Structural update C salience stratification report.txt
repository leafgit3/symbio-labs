Structural Update Report (Track C)
Date: 2026-03-15

Purpose
- Extend structural work after Track A+B with salience stratification.
- Replace flat memory salience with signal-aware dynamic scoring.
- Provide a measurable memory-priority signal that can be surfaced in cycle diagnostics.

What changed

1) New dynamic salience scorer
- Added:
  - src/lib/simulation/salience.ts
- New scorer computes memory salience from:
  - novelty vs recent memories
  - contradiction vs recent feed
  - public/private mismatch (memory vs posted content)
  - threshold-relevant terms (evidence/escalation/containment/promotion/risk)
  - contradiction cues in memory language
  - stance pressure (non-monitor stances slightly upweighted)
  - no-op penalty
- Output range is clamped to 0.20-0.95 and rounded to 2 decimals.

2) Runtime integration (Supabase + in-memory)
- Replaced fixed `salience: 0.65` in both runtime paths with dynamic salience computation.
- Updated file:
  - src/lib/simulation/runCycle.ts

3) Track D dependency alignment
- Track C outputs are now consumed by Track D diagnostics:
  - salienceAvg
  - salienceStdDev
- These are persisted per cycle in `cycle_run_summaries.diagnostics` and shown in Dashboard/History.

Validation
- npm run lint: passed
- npm run build: passed
- Live API verification after redeploy:
  - `/api/agent-memories` shows non-flat salience values
  - latest cycle diagnostics include salience average + stddev
  - example observed values in production run: `salienceAvg 0.82`, `salienceStdDev 0.08`

Expected effect
- Memory salience should now vary across cycles/agents.
- Contradiction-rich and threshold-relevant memories should be prioritized over neutral repetitive records.
